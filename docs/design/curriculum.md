# 営業向け AI教育カリキュラム 設計書（正本）

> 対象職種：営業 ／ 形式：完全リモート・ハンズオン ／ 1セッション60分・週1回・3ヶ月
> ツール：IBM Consulting Advantage（ICA）／ Microsoft Copilot ／ IBM Bob

## 0. この設計書の位置づけと「整合の仕組み」
- 本書は **レベル定義・スケジュール・ゴール分布・業務フローの“正本（Single Source of Truth）”**。台本（`scripts/`）・スライド（`slides/`）・ナビは本書に従う。
- レベルごとの**分刻みの進め方は台本が正**。本書は定義とアウトラインに留め、台本にリンクする（二重持ちを避ける）。
- 整合の保ち方は [設計の整合を保つ仕組み（メンテ規約）](maintenance.md) を参照（変更時チェックリスト＋CIの自動検査）。

## 1. 設計の重心
- 講師がプレゼンするのではなく、**参加者がその場で手を動かして「使えた・やれる」を体感**するのが毎回のゴール。
- 重心は「何を教えるか」より **「60分で何を完了させ、何を持ち帰らせるか（成果物）」**。
- 上達の主エンジンは授業ではなく **宿題＝毎週“実案件”でAIを使う反復** と **横展開（学び合い）**。

## 2. レベル定義と卒業条件
背骨を「ツール別」ではなく **「営業業務 → AIへの渡し方フロー」** で構成（§3）。

| レベル | 一言ゴール | 営業フロー上の担当 | 卒業条件（できること） | 台本 |
|---|---|---|---|---|
| **オンボーディング**（全員必須） | AIに「仕事の素材」を渡す型・マインドセット＋研修の地図 | ②記録する | 会議をTeams録画→Copilotで議事録・報告・次アクションを自力で作れる | [台本](../scripts/onboarding.md) |
| **LV1 チャットAI使い倒し** | 日常業務の大半をチャットAIに渡せる | ①調べる ③書く | リサーチ・要約・メール・下書き・壁打ちを自分のプロンプトで週次常用 | [台本](../scripts/l1.md) |
| **LV2 Bobでデモ** | 提案を「言葉」から「体感」へ | ④見せる | お客様前で見せられる紙芝居デモを Bob で1本作れる | [台本](../scripts/l2.md) |
| **LV3 使い分け** | 案件まるごとAIで回す | ⑤回す（①〜④統合） | 1案件をリサーチ→提案→デモ→フォローまで ICA/Copilot/Bob を使い分けて回せる | [台本](../scripts/l3.md)（改訂予定） |
| **エクストラ**（希望者のみ） | AIを「作る側」に回る | ⑥作る | 業務課題を解くミニアプリ/静的ページを Bob で作り GitHub等で公開してお客様共有できる | （作成予定） |

### レベル診断（毎回これで自分の出る回を選ぶ）
募集時に5問のセルフチェックを配布（Forms）。
1. ICA / Copilot に自分でログインできる
2. 先週、業務でAIを3回以上使った
3. 自分用に保存したプロンプトが3つ以上ある
4. Bobでデモ（紙芝居）を作ったことがある
5. AIの使い方を同僚に教えた／デモをお客様に見せたことがある

- 1〜2中心 → **LV1** ／ 3が「はい」＋デモ未経験 → **LV2** ／ 4〜5「はい」 → **LV3** ／ さらに作りたい → **エクストラ**
- レベルは固定の所属ではなく **その回の入口**。毎回自分で選ぶ。

## 3. カリキュラムの背骨：営業業務 × AIへの渡し方フロー

| 営業の流れ | AIに渡すこと | 主に使うAI | 学ぶレベル |
|---|---|---|---|
| ① 調べる（商談前リサーチ） | 企業・業界・担当者・課題仮説の調査と整理 | ICA / Copilot | LV1 |
| ② 記録する（商談・会議） | 録画→文字起こし→議事録・次アクション | Copilot | オンボーディング |
| ③ 書く（文章作成） | メール・要約・提案下書き・FAQ | Copilot / ICA | LV1 |
| ④ 見せる（提案を体感） | 紙芝居レベルの画面プロト／対話デモ制作 | Bob | LV2 |
| ⑤ 回す（案件まるごと） | ①〜④を最適なAIで一気通貫＋品質管理 | ICA+Copilot+Bob | LV3 |
| ⑥ 作る（業務課題を解く） | 簡単なAIアプリ／静的ページ制作・公開 | Bob | エクストラ |

### 全レベル共通の「AIへの渡し方の型」（毎回繰り返し体に入れる）
```text
① 素材を渡す      … 人が文脈（会議・案件・資料）を持ち込む
② 指示する        … 役割・依頼・出力形式・安全弁（無いことは要確認/事実を作らない）
③ AIに作らせる    … 下書き・整形・デモ・アプリ
④ 人が確認・仕上げ … 事実確認と判断は人間が持つ
（→ ⑤ 再利用資産化：テンプレ／デモ／アプリとして残す）
```
> 型は1つだけ。レベルが上がっても“渡す業務”と“使うAI”が広がるだけなので、習熟が積み上がる。

## 4. 3ヶ月（12週）スケジュール ＝ 混在型 ＋ ゴール分布

**固定進行ではなく、受講生が毎回レベルを自分で選ぶ「混在型」。** 各レベルは期間中に複数回（毎回テーマ＝営業業務を変えて）開く。重みは **前半＝LV1多め／中盤＝LV2多め／後半＝LV3多め**、エクストラは後半に混ぜる。個人の進み方は非線形でOK（例：LV1数回→LV2→LV1戻り→LV2多め→LV3とLV2往復→最後LV3）。

| 週 | その週に開く回（テーマ＝営業業務） | フェーズ重み |
|---|---|---|
| W1 | オンボーディング（全員）＝②記録する ＋ レベル地図・3ヶ月の全体像 | 入口 |
| W2 | LV1-a 調べる（リサーチ） | 前半＝LV1多め |
| W3 | LV1-b 書く（メール・要約・下書き） | 〃 |
| W4 | LV2-a Bob入門・デモ設計（行ける人から） | 〃 |
| W5 | LV1-c 使い倒す（壁打ち・使い分け入門／戻り・復習OK） | 中盤へ |
| W6 | LV2-b デモ制作 | 中盤＝LV2多め |
| W7 | LV2-c デモを磨く・見せ方 | 〃 |
| W8 | LV3-a 使い分けの地図 ＋ エクストラ告知 | 〃 |
| W9 | LV2 実戦（自分の案件でデモ／戻り・復習OK） | 後半へ |
| W10 | LV3-b 案件前半（リサーチ→提案）＋ エクストラ枠(任意) | 後半＝LV3多め |
| W11 | LV3-c 案件中盤（デモ→資料）＋ エクストラ枠(任意) | 〃 |
| W12 | LV3-d 案件後半・統合（フォロー→クロージング＋品質）＋ 成果共有 | 最後はLV3 |

> 開催内訳＝LV1×3／LV2×4／LV3×4／オンボ×1。「その週の回」は目安で、各レベルは複数回あるので早い人は先へ、じっくり派は反復できる。

### 3ヶ月後のゴール分布（W12時点）
| レベル | 目標比率 | 人数イメージ（15名なら） |
|---|---|---|
| LV2（Bobでデモまで） | **1割** | 1〜2名 |
| LV3（使い分けまで） | **7割** | 10〜11名 |
| エクストラ（AIアプリ） | **2割** | 2〜3名 |

> 全員がオンボ〜LV2を通過し大半がLV3へ。LV1止まりは作らない。
> 達成レバー：①渡し方の型が1つで積み上がる ②宿題＝毎週実案件で使う ③LV1を素早く抜けLV2/LV3に時間を回す ④学び合い→教える側（チャンピオン化）。

## 5. 全レベル共通の「60分ハンズオン」フレーム
（ブレイクアウトは最大2回。講師が話すのは合計15分以内、残り45分は参加者が触っている状態を死守）

| 時間 | ブロック | 目的 |
|---|---|---|
| 0:00–0:05 | チェックイン＆今日のゴール宣言＋安全/録画ルール | 期待値合わせ・安心 |
| 0:05–0:14 | ミニデモ（講師は1個だけ実演） | 真似する型を見せる |
| 0:14–0:24 | 写経ワーク（全員が同じ題材で手を動かす） | 「動いた」の成功体験 |
| 0:24–0:39 | ブレイクアウト①（自分の案件に置き換え） | 自分ごと化・教え合い |
| 0:39–0:48 | 全体共有＋フィードバック | 良い例の横展開 |
| 0:48–0:55 | 応用 or ブレイクアウト② | 一段難しい課題 |
| 0:55–1:00 | 保存・自己評価・宿題提示 | 持ち帰り資産化・次週接続 |

## 6. レベル別セッション設計（アウトライン｜詳細は各台本）
各レベルは**毎回テーマ（営業業務）を変える複数回**構成。分刻みの進行は台本が正。

- **オンボーディング**：会議録画→議事録の体験で「素材を渡す」型を体得＋研修の地図（レベル定義・業務フロー・3ヶ月）を共有。→ [台本](../scripts/onboarding.md)
- **LV1（チャットAI使い倒し）**：a 調べる／b 書く／c 使い倒す。リサーチは「確認すべき点を出させ人が裏取り」。→ [台本](../scripts/l1.md)
- **LV2（Bobでデモ）**：a 入門＋初デモ／b 自分の案件で制作／c 磨き・見せ方。紙芝居レベル・画面提示でOK。→ [台本](../scripts/l2.md)
- **LV3（使い分け）**：a 使い分けの地図／b 案件前半／c 案件中盤／d 案件後半・統合＋品質管理。→ [台本](../scripts/l3.md)（改訂予定）
- **エクストラ**：Bobでミニアプリ/静的ページ→GitHub等で公開・共有（LV2デモの“その先”）。（作成予定）

## 7. 完全リモート・ハンズオン運営マニュアル（10〜20名）
「その場でやれる感」は段取りが8割。

- **2名体制**：メインホスト（進行・デモ）＋共同ホスト/TA（チャット監視・BO巡回・つまずき拾い）。L3卒業者を次期TAに育てる。
- **事前セットアップ必須**：前日までに ICA/Copilot/Bob のログイン確認、当日サンプル配布、準備完了をFormsで提出。未完了はTAが個別フォロー。
- **当日**：画面オン推奨（作業画面共有歓迎）／操作中はミュート＋チャット質問／デュアル画面前提／BOは2〜3名固定／「できた✅・詰まった❓」で進捗可視化。

### 会議の録画とトランスクリプト（入力の基本・この会社の前提）
受講者に**会議メモを取る習慣がない**ため「メモを用意して」を前提にしない。
- 会議は **Teamsで録画 → Copilotでトランスクリプト／会議まとめ** を作り、それをAIの素材にする（メモ取り不要）。
- **お客様との打ち合わせは、録画の前に必ず同意を得る**：「記録のために録画させてください。私たちの記録のためだけに使います」。合意を得てから録画。
- 長いトランスクリプトの要約は取り違いが起きうる。重要論点は録画と突き合わせて確認（プロンプトに「無いことは要確認・事実を作らない」を必ず入れる）。

## 8. 効果測定とダッシュボード
- 毎週とる：出席レベル分布（LV1/LV2/LV3/エクストラ）・直前1週間のAI利用回数・宿題提出/成果物数。
- 合否ライン（3ヶ月後）：**LV2 1割／LV3 7割／エクストラ 2割**、週次AI利用率80%以上、エクストラ2〜3人がアプリ公開。
- 可視化：レベル診断Forms → **CSVエクスポートを `dashboard/data/responses.csv` に上書き** → [ダッシュボード](../dashboard/index.md)（生CSVをJSが集計し、到達レベルの分布推移＋目標比較を表示）。Forms設計・運用手順はダッシュボードページが正。**公開サイト内に置く＝氏名・メールを含めない（Formsは匿名設定）**。

### 付録：このカリキュラムが分布目標を実現する論理
人が上達するのは授業でなく**反復された実務利用**。本設計は（a）入口（オンボーディング）で壁を壊し、（b）LV1で日常業務をAIに渡す習慣を作り、（c）LV2で「見せる」武器を持たせ、（d）LV3で案件をまるごと回し、（e）一部をエクストラで“作る側”へ引き上げる。渡し方の型が1つで積み上がるため、混在型でも各自のペースで上のレベルへ到達できる。
