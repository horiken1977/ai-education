# 営業向け AI教育カリキュラム 設計書

> 対象職種：営業 ／ 形式：完全リモート・ハンズオン ／ 1セッション60分・週1回・3ヶ月
> ツール：IBM Consulting Advantage（以下 IBM CA）＋ Microsoft Copilot
> ゴール：3ヶ月で「初心者ゼロ・受講者の半数以上が上級レベル」へ引き上げる

---

## 0. この設計書の前提と読み方

本設計はあなた（運営者）が **講師としてプレゼンするのではなく、参加者がその場で手を動かして「使えた・やれる」を体感する** ことを毎回のゴールに置いています。
したがって設計の重心は「何を教えるか」より **「60分で何を完了させ、何を持ち帰らせるか（成果物）」** にあります。

主要な設計判断（迷ったらこの方針）：

| 論点 | 本設計の判断 | 理由 |
|---|---|---|
| 構成単位 | レベル別に **1本ずつの完成された60分セッション**を作り、3ヶ月で**繰り返し回す** | 条件「各レベル1回実施・繰り返しあり」に対応。新規参加と再参加・昇級を同じ箱で吸収できる |
| 進級の主エンジン | セッションではなく **セッション間の宿題（実務での実使用）** | 60分で人は上達しない。上達するのは「翌週までに自分の案件で3回使った」人。本設計の肝はここ |
| ツール役割分担 | Copilot＝**日常業務（メール・会議・Word/Excel）**、IBM CA＝**提案・調査・エージェント的な深い作業** | 営業の日次摩擦はCopilotで、付加価値はIBM CAで取りに行く |
| 完全リモート対策 | **事前セットアップ必須化＋共同ホスト（TA）常駐＋画面オン推奨** | 60分をログイン・権限トラブルで溶かさないため |

## 1. 3レベルの定義と「卒業条件」

「初心者がいなくなる／半数が上級になる」を測れるよう、各レベルを **行動で定義** します（自己申告ではなく、できることベース）。

### レベル定義表

| レベル | 状態 | このレベルの一言ゴール | 卒業条件（次へ上がる目安） |
|---|---|---|---|
| **L1 初心者** | ほぼ未使用／使ったが慣れていない | 「AIに頼んで仕事の一部が終わった」を体験する | Copilot・IBM CAに自分でログインし、メール下書き等の小タスクを**自力で1つ完了**できる |
| **L2 初級〜中級** | チャットAIには慣れているが、社内・実務への当てはめ方が不明 | 自分の営業業務に**AIを組み込む型（テンプレ）を持つ** | 自分の定常業務3つにAIを使い、**個人プロンプトテンプレを3本**保有・運用している |
| **L3 上級** | 実務活用中。もっと使いこなして生産性・付加価値を上げたい | AIを**仕組み化・横展開**して時間と質を稼ぐ | 複数ステップの業務をAIで型化（再利用資産化）し、**他人に渡せる**状態にできる |

### レベル診断（参加者が毎回これで自分の出席先を選ぶ）

セッション募集時に5問のセルフチェックを配布（Forms等）。
1. Copilot / IBM CA に自分でログインできる（はい/いいえ）
2. 先週、業務でAIを3回以上使った
3. 自分用に保存したプロンプト（指示文）が3つ以上ある
4. AIの結果を自分でレビュー・修正して実務に出せる
5. AIの使い方を同僚に教えたことがある

- 「いいえ」が多い → **L1**　／　1〜2が「はい」中心 → **L2**　／　3〜5が「はい」 → **L3**

> 運営Tips：レベルは固定の所属ではなく **その回の入口**。同じ人がL1→L2→L3と毎月上がっていく前提で設計する。

## 2. 全体スケジュール設計（12週間）

### 採用する型：3週ローテーション × 4サイクル ＋ 前半厚め配分

3ヶ月＝約12回。1週1セッションなので、3レベルを回すローテーションを基本骨格にします。
ただし「初心者を早期に消す」ため **前半は L1/L2 を厚く、後半は L2/L3 を厚く** 配分します。
なお、**全員はまず「オンボーディング」を1回受講**してからレベル別に入ります（新規参加者向けに各サイクルの初回で繰り返し開催）。

| 月 | 週 | 開催レベル | 狙い |
|---|---|---|---|
| **入口** | 各サイクル初回 | **オンボーディング（全員必須）** | マインドセット形成＋「使えそう」の初体験で最初の壁（怖い・分からない）を超える |
| **1ヶ月目** | W1 | **L1** | 全員に入口を。最大母数を初体験させる |
| | W2 | **L1**（再）or L2 | 初回欠席者の救済＋早い人をL2へ |
| | W3 | **L2** | 「自分の業務に当てる」へ移行開始 |
| | W4 | **L1**（再）/ L2 | 月内の新規・出戻りを吸収 |
| **2ヶ月目** | W5 | **L2** | 中核フェーズ。型づくりを量産 |
| | W6 | **L3** | 早い層を上級へ引き上げ開始 |
| | W7 | **L2** | L1卒業組の受け皿 |
| | W8 | **L3** | 上級者の比率を上げる |
| **3ヶ月目** | W9 | **L2 / L1（最終救済）** | 残る初心者をここでゼロに |
| | W10 | **L3** | 仕組み化・横展開 |
| | W11 | **L3** | 上級スキルの定着 |
| | W12 | **L3 ＋ 成果共有会** | 半数以上を上級に。全社共有でモメンタム化 |

> このグリッドは**固定ではなく診断結果で毎月チューニング**します。L1出席者が減らなければL1を足し、L3希望が増えたらL3を増やす。KPI（§6）を見て翌月の配分を決めるのが正しい運用です。

### 進級パイプラインのイメージ

```
新規 ──▶ オンボーディング（全員必須・マインドセット＋壁超え）
          │
          ▼
        L1（初体験・自力で1タスク完了）
          │  宿題：今週、業務で3回使う
          ▼
        L2（自分の業務に型を作る・テンプレ3本）
          │  宿題：定常業務3つにテンプレ運用
          ▼
        L3（仕組み化・横展開・他人に渡す）
          │  宿題：チームに1資産を共有/布教
          ▼
       社内チャンピオン（次期セッションの共同ホスト候補）
```

## 3. 全レベル共通の「60分ハンズオン」フレーム

毎回このリズムで進めると、リモートでも崩れません（ブレイクアウトは最大2回）。

| 時間 | ブロック | 形式 | 目的 |
|---|---|---|---|
| 0:00–0:05 | **チェックイン＆今日のゴール宣言** 「終わったらこれができる」を1文で約束 | 全体 | 期待値合わせ・安心感 |
| 0:05–0:12 | **ミニデモ（講師は1個だけ実演）** | 全体 | 真似する型を見せる。説明は最小限 |
| 0:12–0:22 | **一緒にやる①（写経ワーク）** 全員が同じ題材で手を動かす | 各自操作 | 「動いた」の最初の成功体験 |
| 0:22–0:37 | **ブレイクアウト①：ペアワーク** 自分の案件に置き換えて試す | BO 2–3人 | 自分ごと化。教え合いで定着 |
| 0:37–0:46 | **全体共有＋講師フィードバック** 2–3名に画面共有してもらう | 全体 | 良い例の横展開・つまずき解消 |
| 0:46–0:55 | **応用ワーク or ブレイクアウト②** | 個人/BO | 一段難しい・実務に近い課題 |
| 0:55–1:00 | **成果の保存・自己評価・宿題提示** | 全体 | 持ち帰り資産化と次週への接続 |

**設計原則**
- 講師の話は**合計15分以内**。残り45分は参加者が触っている状態を死守。
- 1セッション＝**1つの持ち帰り成果物**（下書きメール／テンプレ／ワークフロー等）。
- 「うまくいかない」も学び。**失敗例を全体共有**してプロンプト改善を体感させる。

## 4. レベル別 セッション設計（各60分・営業文脈）

### オンボーディング（全員必須・L1の前に1回）「AIは怖くない。一緒に“身につけて”いこう」

> **狙い**：知識を教える回ではなく、**マインドセットを“体験を通して”渡す**回。冒頭10分の説明だけでは納得は得られないので、**先に手を動かして「なるほど、これは使えそう」を体感**させ、「AIってよく分からない・怖い・信じられない」という最初の壁を超える。L1の内容と重複してOK。

**この回で必ず届ける6つのメッセージ**（すべて“体験に紐づけて”短く）
1. AIを使えることはこれからの必須スキル。**学び続け・変わり続ける**行動を取ろう。
2. ゴールは「わかる」でも「できる」でもなく、その先の **「身につく」**。
3. これは私が教える場ではない。**私も一緒に学び・身につけ・上達していく**。受講者どうしの**学び合い**が何より大事。
4. レベルは3段階。**どこから受けてもOK**。自分のレベル感に正直に。
5. **カリキュラムは途中で変わる**。AIの進化が桁違いに速いから（変化に乗れること自体がスキル）。
6. （上記すべてを）**手を動かして腑に落とす**。説明より体験。

**わかる → できる → 身につく の階段（この研修の背骨）**

| 段階 | 状態 | どこで効くか |
|---|---|---|
| **わかる** | AIで何ができるか腑に落ちる | オンボーディング前半 |
| **できる** | 手伝ってもらえば一度できた | オンボーディング後半・L1 |
| **身につく** | 考えなくても普段使いになる | **毎週の反復・宿題（ここが本体）** |

- **今日のゴール宣言**：「終わったら、①AIを自分の手で動かして“使えそう”と感じ、②この研修への自分なりの向き合い方（自分ごと）を1つ言葉にできる」
- **使うツール**：Copilot（入口が易しい）。

| 時間 | 内容 |
|---|---|
| 0:00–0:05 | チェックイン。「今日も研修全体も“私が教える場”ではなく一緒に学ぶ場」宣言＋安全ルール |
| 0:05–0:12 | ファーストコンタクト：説明より先に、全員が1プロンプト（要約／説明／言い換えの3択）を実行 |
| 0:12–0:20 | 体験の振り返り→マインドセット①②③を体験に紐づけて。階段（わかる/できる/身につく）提示 |
| 0:20–0:35 | BO①：ペアでL1ライト（会議トランスクリプト→議事録・報告・次アクション）。「自分でやったら何分?」を会話 |
| 0:35–0:43 | 全体共有＋「私も学んでいる」を体現（弱さを見せる）。自分ごとへの橋渡し |
| 0:43–0:50 | 地図：3レベル説明＋「どこからでもOK」＋「変わる前提」。その場で5問セルフ診断 |
| 0:50–0:57 | BO②：自分ごと宣言（なぜ自分に必要か＋今週の小さな1歩）をペアで宣言し合う |
| 0:57–1:00 | クローズ：階段を再掲。宿題（今週1回使う＋次回は自分のレベルへ） |
- **持ち帰り成果物**：①AIで作った成果（整形済みメモ等）／②「自分ごと宣言」（なぜ＋今週の1歩）
- **宿題（次週まで）**：今週、業務で1回でいいのでAIを使う。次回は自分のレベルの回に出席。
- **詳細台本**：[オンボーディング台本](../scripts/onboarding.md)

### L1：初心者セッション「AIに頼んで、仕事の一部を終わらせる」

> **設計原則（重要）**：L1は「AIに文脈を生成させる」タスク（例：顧客への返信メールを書かせる）を**避ける**。
> 背景を知らないAIの生成物は当たり障りがなく、結局その修正で逆に遅くなり、初心者の最初の体験を台無しにする。
> 代わりに **「人が中身を持ち込み、AIは整形・構造化に徹する」タスク** を入口にする。
> → 採用：**Teams会議の録画トランスクリプトを、整った議事録・上司報告・次アクション一覧に整形する**。
> **前提（この会社）**：受講者に会議メモを取る習慣がなく「メモを用意して」は高いハードル。だから **会議はTeamsで録画し、Copilotのトランスクリプトを“素材”にする**ことを徹底して推す（メモ取りは不要）。中身は会議そのものなのでハルシネーションが起きにくく、毎日の雑務の時短を確実に体感できる。
> ※当面は触れないが、本来は重要論点を“その場でタイプ”する方が速く正確。できる人が少ないため、まずは録画→トランスクリプトを基本にする。

- **今日のゴール宣言**：「終わったら、Teams会議の録画トランスクリプトを、整った議事録・上司報告・次アクション一覧に“自力で”整形できる（＝メモを取らなくていい）」
- **使うツール**：まず Copilot／Teams（録画・文字起こし）。最後に IBM CA を1分だけ触れて存在を知る。
- **録画の同意ルール**：お客様との打ち合わせは、録画前に必ず「記録のために録画させてください。私たちの記録のためだけに使います」と断り、**合意を得てから録画**する。

| 時間 | 内容（営業の具体題材） |
|---|---|
| 0:00–0:05 | チェックイン。「今日は会議の“録画”を“そのまま出せる議事録”に変えて終わります」＋安全ルール＋録画同意ルール |
| 0:05–0:12 | デモ：会議トランスクリプトを貼って「①議事録 ②上司への報告 ③次アクション一覧」の3点に整形 |
| 0:12–0:22 | 写経ワーク：配布の架空トランスクリプトを貼り、**3点セットへの整形**を全員が実行 |
| 0:22–0:37 | BO①：ペアで「自分の直近のTeams会議トランスクリプト（無ければ配布サンプル）」を整形し、見せ合う |
| 0:37–0:46 | 共有：2名の画面。**プロンプトの型**を板書「①役割 ②素材（文字起こし） ③依頼 ④出力形式」 |
| 0:46–0:55 | 応用：「箇条書きを文章に」「報告は3行で」「ToDoに期日列を足して」と**出力形式を操る**体験 |
| 0:55–1:00 | 保存：今日の良かったプロンプトをOneNoteへ。宿題提示。IBM CAのログインだけ確認 |
- **持ち帰り成果物**：会議トランスクリプトから作った**議事録＋上司報告＋次アクション一覧**（今日そのまま使える）
- **宿題（次週まで）**：次の会議をTeamsで録画（顧客には事前同意）→ Copilotでトランスクリプト→AIで整形。1件でも共有・提出できたら成功。

### L2：初級〜中級セッション「自分の営業業務に、AIの型を作る」

- **今日のゴール宣言**：「終わったら、自分の定常業務に使える**プロンプトテンプレを3本**持って帰る」
- **使うツール**：Copilot（M365内：Outlook/Teams/Word/Excel）＋ IBM CA（調査・提案ドラフト）。

|---|---|

## 5. 完全リモート・ハンズオン運営マニュアル（10〜20名）

「その場でやれる感」を出すには**運営の段取りが8割**です。

### 役割分担（最低2名体制）
- **メインホスト**：進行・デモ・時間管理。
- **共同ホスト（TA）**：チャット監視・個別トラブル対応・BO巡回・つまずき拾い。
  - ※L3卒業者を次期TAに育てると、運営が自走し横展開も進む（一石二鳥）。

### 事前セットアップ（最重要・60分を守る命綱）
募集メールに同梱し、**前日までに完了を必須化**：
1. Copilot / IBM CA に各自ログインできることを確認（権限・ライセンス）
2. 当日使うサンプルファイル（メール文・会議文字起こし・案件リスト）を配布
3. 「準備完了チェック」を前日Forms提出 → 未完了者はTAが個別フォロー

### 会議の録画とトランスクリプト（入力の基本・この会社の前提）
受講者には**会議メモを取る習慣がない**ため、「メモを用意して」を前提にしない。会議の入力は次を基本にする。
- **会議はTeamsで録画 → Copilotでトランスクリプト／会議まとめを生成 → それをAIの素材にする**（メモ取りは不要）。L1・オンボーディングで徹底して推す。
- **お客様との打ち合わせは、録画の前に必ず同意を得る**。言い方の例 →「記録のために録画させてください。私たちの記録のためだけに使います」。合意を得てから録画する。
- 社内会議も録画を基本にしてOK。長いトランスクリプトの要約は取り違いが起きうるので、重要論点は録画と突き合わせて確認（プロンプトに「無いことは“要確認”、事実を作らない」を必ず入れる）。
- ※当面は触れないが、本来は重要論点を“その場でタイプ”する方が速く正確。できる人が増えたら段階的に推奨する。

### 当日の進め方
- **画面オン推奨**（顔でなく作業画面の共有を歓迎＝つまずきが見える）。
- 操作中は**ミュート＋チャットで質問**。TAがリアルタイム回答。
- **デュアル画面前提**：講師画面を見ながら自分のツールを操作できるよう案内。
- BOは**2〜3名固定ペア**。少人数の方が短時間で教え合いが起きる。
- **「できた」スタンプ運用**：各ワーク完了時にチャットへ「✅」。進捗が一目で分かり、置いてけぼりを防ぐ。

### よくある事故と対策
| 事故 | 対策 |
|---|---|
| ログイン・権限で15分溶ける | 事前セットアップ必須化＋TAが開始前10分に待機部屋でフォロー |
| 1人が遅れて全体停滞 | TAがBO/DMで個別救済。全体は止めない |
| 社外秘・個人情報の入力 | 冒頭1分で**入力禁止情報のルール**を毎回提示（伏字・架空データを使う） |
| 静まり返る | 「できた人✅」「詰まってる人❓」をチャットで定期的に取る |

### 付録：このカリキュラムが「半数上級化」を実現する論理
人が上級になるのは授業ではなく**反復された実務利用**による。本設計は
（a）入口を限界まで易しくして初体験の離脱を防ぎ（L1）、
（b）自分の業務に型を作らせて利用を定着させ（L2）、
（c）他人に渡せる資産化＝教える側に回す（L3）、
という**「使う→型化→教える」の上昇スパイラル**を、宿題と横展開で毎週回す。
教える側に回った人数が増えるほど上級比率は加速し、3ヶ月での「半数以上が上級」が現実的になる。
