# 講義モード設計（Webスライド投影 ＋ アクセス制御）大枠

> 目的：PPTを作らず、**設計済みのWebを「PPTのように1画面ずつ切り替えて投影」**しながら話せるようにする。
> あわせて **ダッシュボードは運営者だけ／講義スライドは受講者も** 閲覧できるよう制御する。
> 確定方針：スライド＝**Marp**、ダッシュボード＝**M365で非公開**。

---

## 1. 全体像（2つの“面”に分ける）

```
┌─ 公開（受講者も閲覧可・ログイン不要）──────────────────────────┐
│  GitHub Pages（現行 https://horiken1977.github.io/ai-education/）   │
│   ├─ 講義スライド（Marp）        … 投影＆受講者の復習用            │
│   ├─ 台本（運営者の手元用・機密ではないので公開のまま）             │
│   ├─ 配布教材・テンプレ                                            │
│   └─ 設計ドキュメント                                              │
└──────────────────────────────────────────────────────────────────┘

┌─ 非公開（運営者だけ・Microsoftログイン）─────────────────────────┐
│  Microsoft 365                                                     │
│   └─ レベル分布ダッシュボード（Power BI / Excel on SharePoint）      │
│       ※公開サイトからは撤去し、ここに移設                          │
└──────────────────────────────────────────────────────────────────┘
```

**なぜこの形か（アクセス制御の正攻法）**
公開GitHub Pages（個人アカウント）は **ページ単位の閲覧制御ができない**（できるのはEnterprise Cloudのみ）。
そこで「見せたいもの＝公開」「隠したいもの＝公開サイトの外（M365）」と **置き場所で分離** する。
- 講義スライド＝公開Pages＝受講者はログイン不要で閲覧可。
- ダッシュボード＝M365＝Microsoftログイン＋運営者の権限でのみ閲覧可＝**本当に“私だけ”**。

---

## 2. スライド方式：Marp（Markdown → スライド）

### 2-1. 考え方：台本とスライドを分ける
| 種類 | 役割 | 投影するか | 置き場所 |
|---|---|---|---|
| **台本（scripts/）** | 運営者の手元用の詳細な進行（セリフ・操作・チャット・TA・つまずき） | しない（手元画面） | 既存 `docs/scripts/` |
| **スライド（slides/）** | 受講者が見る“投影画面”。1画面1メッセージ・大きな文字 | する（投影） | 新規 `docs/slides/` |

- 1画面ずつ切り替え（矢印キー）・フルスクリーン（F）＝**PPTと同じ操作感**。スクロールは使わない。
- **2画面運用**：投影＝スライド／手元＝台本。台本を見ながらスライドを送る。

### 2-2. デッキ構成（1セッション＝1デッキ・60分フレームに対応）
各セッション（オンボーディング／L1／L2／L3）に1デッキ。スライドは下の型で起こす：

1. タイトル＋**今日のゴール宣言**
2. 安全ルール＋**録画の同意ルール**
3. ミニデモの見出し＋**コピー用プロンプト（大きく）**
4. 写経ワークの指示
5. **ブレイクアウト①**の手順（大きく）
6. 全体共有／プロンプトの「型」
7. 応用ワーク／ブレイクアウト②
8. まとめ・宿題・次回

> 1スライド＝1アイデア。文字は大きく最小限。詳細なセリフは投影せず台本側に置く。

### 2-3. Marp の作り方（イメージ）
`docs/slides/l1.md` の冒頭と区切り：
```
---
marp: true
theme: default
paginate: true
---

# L1：会議の“録画”を議事録に
**今日のゴール：録画トランスクリプトを議事録・報告・次アクションに整える**
```

## 3. ダッシュボードのアクセス制御：M365へ移設

### 3-1. 変更点
- 公開リポジトリから **`docs/dashboard/` を撤去**（サンプルCSV `levels.csv` も削除＝公開データを残さない）。
- 公開サイトには「**ダッシュボードはM365（要ログイン・運営者限定）**」という案内だけ残す（リンク先は権限がない人には開けない）。
- ダッシュボード実体は **Power BI または Excel(SharePoint)**：レベル診断Forms → Excel自動連携 → レベル判定 → グラフ。共有範囲＝**自分のみ**。

### 3-2. これで満たされること
- 受講者：講義スライド（公開）は見られる。ダッシュボードのURLを知っていても、M365の権限がなければ**開けない**。
- 運営者：Microsoftログインでダッシュボードを閲覧。**本当に“私だけ”**。
- 以前のM365基盤の決定と整合（むしろ本来の置き場所に戻す）。

## 5. 当日の投影オペレーション
1. 投影画面：スライドHTMLを開き、**F でフルスクリーン**、**矢印キーでページ送り**（スクロールしない）。
2. 手元画面：[台本](../scripts/index.md) を開き、セリフ・操作・チャット文を見ながら進行。
3. ハンズオン：スライドに**コピー用プロンプトを大きく表示** → 受講者は見ながら自分のCopilotで操作。
4. 受講者は同じスライドURLを各自開けば手元でも追える（任意・後日の復習にも）。
