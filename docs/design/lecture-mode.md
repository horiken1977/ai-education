# 講義モード設計（Webスライド投影）大枠

> 目的：PPTを作らず、**設計済みのWebを「PPTのように1画面ずつ切り替えて投影」**しながら話せるようにする。
> 確定方針：スライド＝**Marp**。ダッシュボードは機密ではないため**公開サイト内に置く**（講義ページとは**ディレクトリを分けるだけ**：`/slides/` と `/dashboard/`）。

---

## 1. 全体像

```
┌─ 公開 GitHub Pages（受講者も閲覧可・ログイン不要）────────────────┐
│  https://horiken1977.github.io/ai-education/                       │
│   ├─ 講義スライド（Marp）   /slides/     … 投影＆受講者の復習用     │
│   ├─ 台本・配布教材・テンプレ・設計ドキュメント                     │
│   └─ ダッシュボード         /dashboard/  … レベル分布の推移（公開）  │
└──────────────────────────────────────────────────────────────────┘
```

**考え方（ディレクトリで分けるだけ）**
- 講義スライドとダッシュボードは **同じ公開サイト内の別ディレクトリ**（`/slides/` と `/dashboard/`）に置く。
- ダッシュボードは個人が特定できる生データを載せない前提で **機密ではない**ため、公開のまま運用する（アクセス制御はしない）。
- ※もし将来「運営者だけ」に厳密に隠したくなったら、その時にM365（Power BI等）へ移す選択肢もある（公開GitHub Pagesはページ単位の閲覧制御ができないため）。

---

## 2. スライド方式：Marp（Markdown → スライド）

### 2-1. 考え方：台本とスライドを分ける
| 種類 | 役割 | 投影するか | 置き場所 |
|---|---|---|---|
| **台本（scripts/）** | 運営者の手元用の詳細な進行（セリフ・操作・チャット・TA・つまずき） | しない（手元画面） | 既存 `docs/scripts/` |
| **スライド（slides/）** | 受講者が見る“投影画面”。1画面1メッセージ・大きな文字 | する（投影） | 新規 `docs/slides/` |

- 1画面ずつ切り替え（矢印キー）・フルスクリーン（F）＝**PPTと同じ操作感**。スクロールは使わない。
- **2画面運用**：投影＝スライド／手元＝台本。台本を見ながらスライドを送る。

### 2-2. デッキ構成（1セッション＝1デッキ・60分フレームに対応）
各セッション（オンボーディング／L1／L2／L3）に1デッキ。スライドは下の型で起こす：

1. タイトル＋**今日のゴール宣言**
2. 安全ルール＋**録画の同意ルール**
3. ミニデモの見出し＋**コピー用プロンプト（大きく）**
4. 写経ワークの指示
5. **ブレイクアウト①**の手順（大きく）
6. 全体共有／プロンプトの「型」
7. 応用ワーク／ブレイクアウト②
8. まとめ・宿題・次回

> 1スライド＝1アイデア。文字は大きく最小限。詳細なセリフは投影せず台本側に置く。

### 2-3. Marp の作り方（イメージ）
`docs/slides/l1.md` の冒頭と区切り：
```
---
marp: true
theme: default
paginate: true
---

# L1：会議の“録画”を議事録に
**今日のゴール：録画トランスクリプトを議事録・報告・次アクションに整える**

---

## まずコピーするプロンプト
（大きく1枚で表示。受講者はこれを見て自分のCopilotで実行）
```
- `---` がスライドの区切り＝PPTのページ切り替え。
- 出力：HTML（投影用）。必要なら **PDF / PPTX も同じソースから出力可**（保険）。

### 2-4. ビルド／公開（GitHub Actions）
- 現行の `deploy.yml` に **Marpビルド工程を追加**：`docs/slides/*.md` → `site/slides/*.html`。
- MkDocsビルドの出力（`site/`）と**同じ場所に出力**し、まとめてPagesへ公開。
- 受講者・運営者とも `…/ai-education/slides/l1.html` のようなURLで開ける。MkDocsのトップ／ナビからリンク。

---

## 3. ダッシュボード：公開サイト内の別ディレクトリ
- ダッシュボードは公開GitHub Pages内の **`/dashboard/`** に置く（Chart.jsでレベル分布の推移を表示）。
- 講義スライド（`/slides/`）とは**ディレクトリを分けるだけ**。アクセス制御はしない（機密ではないため）。
- データは Forms→CSVエクスポートを `docs/dashboard/data/responses.csv` にコミット（JSが集計）。**氏名・メールは含めない（Formsは匿名設定）**。
- ※「本当に運営者だけ」に隠したい場合のみM365へ移す（今回は不要と判断）。

---

## 4. リポジトリ構成（更新後イメージ）

```
ai-education（公開リポジトリ / 公開Pages）
├─ docs/
│  ├─ index.md
│  ├─ slides/                 ← Marpソース（onboarding/l1/l2/l3）… /slides/ で公開
│  ├─ scripts/                ← 台本（手元用）
│  ├─ materials/ templates/
│  ├─ dashboard/              ← ダッシュボード（公開・別ディレクトリ）… /dashboard/
│  └─ design/                 ← 設計（本書 lecture-mode.md を含む）
├─ mkdocs.yml
└─ .github/workflows/deploy.yml   ← MkDocs + Marp を同じ site/ にビルドしPages公開
```

## 5. 当日の投影オペレーション
1. 投影画面：スライドHTMLを開き、**F でフルスクリーン**、**矢印キーでページ送り**（スクロールしない）。
2. 手元画面：[台本](../scripts/index.md) を開き、セリフ・操作・チャット文を見ながら進行。
3. ハンズオン：スライドに**コピー用プロンプトを大きく表示** → 受講者は見ながら自分のCopilotで操作。
4. 受講者は同じスライドURLを各自開けば手元でも追える（任意・後日の復習にも）。

---

## 6. 次の一手
1. `deploy.yml` に Marp ビルド工程を追加（PoC：1デッキで投影確認）。← **実施済み（L1）**
2. **残り3デッキを設計から起こす**（オンボーディング／L2／L3）。スライドの型は §2-2。
3. ダッシュボードは公開サイト内 `/dashboard/` のまま運用（M365移設はしない）。

> スライド本体（L1）とMarpビルドは実装済み。残デッキは順次作成。
