# ダッシュボード（M365・非公開／運営者のみ）

> レベル分布の推移（初心者↓・上級↑）を可視化する **運営者専用** ダッシュボード。
> **公開サイトには置かない**（受講者の進捗データは非公開）。実体は Microsoft 365 に置き、共有範囲を「**自分のみ**」にする。

## なぜM365なのか（アクセス制御）
公開GitHub Pages（個人アカウント）は**ページ単位の閲覧制御ができない**。受講者の進捗データを守るため、ダッシュボードは公開サイトから外し、**Microsoftログイン＋自分の権限でのみ開けるM365**に置く。これで「本当に私だけが見られる」が成立する。
（公開リポジトリにあった Chart.js 版ダッシュボード `docs/dashboard/` は、この要件のため撤去済み。経緯は [講義モード設計](lecture-mode.md) §3。）

## データの流れ
```text
レベル診断（Microsoft Forms・5問）
   └→ 回答を Excel に自動連携（Forms「Excelで開く」）
        └→ レベル判定列＋週次集計（week, L1, L2, L3）
             └→ 可視化（Excelグラフ or Power BI）＝共有しない（自分のみ）
```

## 作り方A：Excelだけで完結（最短・まずこれ）
1. **Forms**：5問のレベル診断を作成（[カリキュラム設計書](curriculum.md) §1）。
2. Forms回答 →「**Excelで開く**」で回答ブックを自分の OneDrive（個人）に保存。
3. 集計シートで週ごとに L1/L2/L3 の人数を集計。
   - レベル自動判定（Q1〜Q5を1/0で集計）：`=IF(合計<=1,"L1",IF(合計<=3,"L2","L3"))`
   - 週次表の形：`week, L1, L2, L3`（撤去した PoC の levels.csv と同じ形）
4. 集計表から **折れ線グラフ**（レベル分布の推移）を作成。目標ライン＝L1を0へ／L3を50%超へ。
5. **共有しない**＝自分の OneDrive 個人領域に保存＝自分のみ閲覧。

## 作り方B：Power BI（見栄え・自動更新したいとき）
1. Power BI Desktop / サービスで、上の Excel（または Forms）をデータソースに接続。
2. レベル分布の推移（折れ線）＋目標ライン（L1=0、L3≧50%）をレポート化。
3. 保存先＝**自分のマイワークスペース**（共有・発行しない）＝自分専用で非公開。

## アクセス制御チェック（重要）
- [ ] 保存先は自分の OneDrive 個人 or 自分だけがメンバーのワークスペース
- [ ] 「リンクを知っている全員」での共有に **していない**
- [ ] 受講者・他メンバーに共有 **していない**
- [ ] 公開GitHubサイトにデータ（CSV等）を **置いていない**

## 受講者・経営への見せ方
- 公開サイトにダッシュボードは置かない。
- 成果（初心者↓・上級↑）は **成果共有会（W12）で要約だけ** 口頭／スライドで共有する。
- 個人が特定できる生データは出さない。

---

### 設定で詰まったら（運営者メモ）
Forms / Excel / Power BI はすべて自分のM365テナント内の操作です。ログインは各サービスのSSOで行います。
本リポジトリ（GitHub）側からM365を操作することはできないため、上記手順は**運営者がM365上で実施**してください。
