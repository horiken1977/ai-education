# カリキュラム改訂版（レベル再定義 ＋ 3ヶ月プラン）

> 営業職向け ／ 完全リモート・ハンズオン ／ 1セッション60分・週1回・3ヶ月
> ツール：IBM Consulting Advantage（ICA）／ Microsoft Copilot ／ IBM Bob
> **背骨を「ツール別」から「営業の業務フロー → AIへの渡し方」に組み替え、各レベルの期待値を引き上げた版。**

---

## 0. 何を変えたか（要点）
- **旧L1（会議録画→議事録）は「オンボーディング」に吸収**。全員の入口に統一。
- レベルを一段引き上げて再定義：
  - **LV1**：チャットAI（ICA / Copilot）を**使い倒せる**
  - **LV2**：**IBM Bob でデモを作り、お客様に見せられる**
  - **LV3**：業務に応じて **ICA / Copilot / Bob を使い分けられる**
  - **エクストラ（希望者のみ）**：**Bob で簡単なAIアプリを作れる**
- カリキュラムの軸を **「営業の業務を、どんな流れでAIに渡すか」** に統一（§2）。
- 進め方は固定進行ではなく、**受講生が毎回レベルを自分で選ぶ「混在型」**（前半＝LV1多め／中盤＝LV2多め／後半＝LV3多め、エクストラも混ぜる）（§3）。
- **オンボーディングで、レベル定義・業務フローの背骨・3ヶ月の全体像を共有**する。
- 3ヶ月後のゴール分布：**LV2 1割／LV3 7割／エクストラ 2割（2〜3人）**。

> ※「IBM Bob」は、デモ制作・簡単なAIアプリ制作ができるツールとして記述。具体的な操作手順は台本作成時に確認する。

## 1. 新レベル定義と卒業条件

| レベル | 一言ゴール | 営業フロー上の担当 | 卒業条件（できること） |
|---|---|---|---|
| **オンボーディング**（全員必須・旧L1吸収） | AIに「仕事の素材」を渡す型・マインドセット ＋ 研修の地図 | ②記録する | 会議を録画→Copilotで議事録等を自力で作れる／レベル定義・業務フロー・3ヶ月の流れが分かる |
| **LV1：チャットAI使い倒し** | 日常業務の大半をチャットAIに渡せる | ①調べる ③書く | リサーチ・要約・メール・下書き・壁打ちを自分のプロンプトで**週次で常用** |
| **LV2：Bobでデモ** | 提案を「言葉」から「体感」へ | ④見せる | お客様前で見せられるデモを Bob で**1本作れる** |
| **LV3：使い分け** | 案件まるごとAIで回す | ⑤回す（①〜④統合） | 1案件をリサーチ→提案→デモ→フォローまで、ICA/Copilot/Bobを**使い分けて回せる** |
| **エクストラ**（希望者のみ） | AIを「作る側」に回る | ⑥作る | 業務課題を解くミニAIアプリを Bob で**1つ公開** |

## 2. カリキュラムの背骨：営業業務 × AIへの渡し方フロー

営業の一連の流れを「どの業務を・どのAIに・どう渡すか」で並べ、各レベルがどの段を担うかを対応づける。

| 営業の流れ | AIに渡すこと | 主に使うAI | 学ぶレベル |
|---|---|---|---|
| ① 調べる（商談前リサーチ） | 企業・業界・担当者・課題仮説の調査と整理 | ICA / Copilot | **LV1** |
| ② 記録する（商談・会議） | 録画→文字起こし→議事録・次アクション | Copilot | **オンボーディング** |
| ③ 書く（文章作成） | メール・要約・提案下書き・FAQ | Copilot / ICA | **LV1** |
| ④ 見せる（提案を体感） | デモ・プロトの制作 | Bob | **LV2** |
| ⑤ 回す（案件まるごと） | ①〜④を最適なAIで一気通貫＋品質管理 | ICA+Copilot+Bob | **LV3** |
| ⑥ 作る（業務課題を解く） | 簡単なAIアプリ制作 | Bob | **エクストラ** |

### 全レベル共通の「AIへの渡し方の型」（これを繰り返し体に入れる）
```
① 素材を渡す      … 人が文脈（会議・案件・資料）を持ち込む
② 指示する        … 役割・依頼・出力形式・安全弁（要確認/事実を作らない）
③ AIに作らせる    … 下書き・整形・デモ・アプリ
④ 人が確認・仕上げ … 事実確認と判断は人間が持つ
（→ ⑤ 再利用資産化：テンプレ／デモ／アプリとして残す）
```
> この型は1つだけ。レベルが上がっても“渡す対象（業務）”と“使うAI”が広がるだけ。だから積み上がる。

## 3. 3ヶ月（12週）ざっくり設計

**固定進行ではなく、受講生が毎回レベルを自分で選ぶ「混在型」。** 各レベルは期間中に複数回開く（毎回テーマ＝営業業務を変える）。
全体の重みは **前半＝LV1多め／中盤＝LV2多め／後半＝LV3多め**、エクストラは後半に混ぜる。
だから進み方は人によって非線形になる（例：LV1を数回 → LV2 → LV1に戻る → LV2を多め → LV3とLV2を往復 → 最後はLV3）。

各レベルは毎回テーマを変えるので、複数回出ても重複しない：

- **LV1**：a 調べる（リサーチ）／b 書く（メール・要約・下書き）／c 使い倒す（壁打ち・ICA/Copilot使い分け入門）
- **LV2**：a Bob入門・デモ設計／b デモ制作／c 磨き・見せ方
- **LV3**：a 使い分けの地図／b 案件前半（リサーチ→提案）／c 案件中盤（デモ→資料）／d 案件後半・統合（フォロー→クロージング＋品質）
- **エクストラ**：Bobでミニアプリ（設計→制作→公開）

| 週 | その週に開く回（テーマ＝営業業務） | フェーズ重み |
|---|---|---|
| **W1** | オンボーディング（全員）＝②記録する ＋ レベル地図・3ヶ月の全体像 | 入口 |
| **W2** | LV1-a 調べる（リサーチ） | 前半＝LV1多め |
| **W3** | LV1-b 書く（メール・要約・下書き） | 〃 |
| **W4** | LV2-a Bob入門・デモ設計（行ける人から挑戦） | 〃 |
| **W5** | LV1-c 使い倒す（壁打ち・使い分け入門／戻り・復習OK） | 中盤へ |
| **W6** | LV2-b デモ制作 | 中盤＝LV2多め |
| **W7** | LV2-c デモを磨く・見せ方 | 〃 |
| **W8** | LV3-a 使い分けの地図 ＋ エクストラ告知 | 〃 |
| **W9** | LV2 実戦（自分の案件でデモ／復習・戻りOK） | 後半へ |
| **W10** | LV3-b 案件前半（リサーチ→提案）＋ エクストラ枠(任意) | 後半＝LV3多め |
| **W11** | LV3-c 案件中盤（デモ→資料）＋ エクストラ枠(任意) | 〃 |
| **W12** | LV3-d 案件後半・統合（フォロー→クロージング＋品質）＋ 成果共有 | 最後はLV3 |

> 「その週の回」は目安。受講生は診断結果と自分のレベル感で「出る／復習する／先へ行く」を選ぶ。各レベルが複数回あるので、早い人は先へ、じっくり派は同レベルを反復できる。
> 集計：開催回内訳＝LV1×3／LV2×4／LV3×4／オンボ×1。前半LV1・中盤LV2・後半LV3に重みが寄る。
> （任意運営）出席が割れたら、1回の中でブレイクアウトを2レベル並行にしてもよい。

### エクストラ（希望者のみ・並行/追加枠）
- **開講タイミング**：W8で告知し、**W10–11に任意枠**（本流のLV3と並行／オフィスアワー形式）。LV2を早く卒業した先行層向け。
- **内容**：自分・チームの業務課題を1つ選び、Bobで **ミニAIアプリ**を 設計→制作→公開。2〜3人想定。
- **アウトプット**：チームで使える小さなアプリ1つ（＝AIを“作る側”の体験）。

## 4. 3ヶ月後のゴール分布と、その達成方法

### 目標分布（W12時点）
| レベル | 目標比率 | 人数イメージ（15名なら） |
|---|---|---|
| LV2（Bobでデモまで） | **1割** | 1〜2名（ペースがゆっくりな層） |
| LV3（使い分けまで） | **7割** | 10〜11名（中核） |
| エクストラ（AIアプリ） | **2割** | 2〜3名（先行層） |

> 全員がオンボーディング〜LV2を通過し、大半がLV3へ。先行層はエクストラへ。LV1止まりは作らない設計。

### この分布を実現するレバー（ここが肝）
1. **渡し方の型は1つ**（§2）。レベルが上がっても同じ型の応用なので、習熟が積み上がり加速する。
2. **宿題＝毎週“実案件”でAIを使う**。上達するのは授業ではなく実務での反復。毎回「自分の案件でやる」を課す。
3. **LV1を素早く抜ける設計**。日常業務（調べる・書く）は到達が速いので、早めにLV2/LV3へ時間を回す。
4. **学び合い**：ブレイクアウトで見せ合い、先行者が教える側へ（チャンピオン化）。教えると定着し、LV3比率が上がる。
5. **エクストラは“引き上げ”の象徴**：2〜3人が作る側に回る姿が、全体の到達イメージを引き上げる。

### 進捗の見える化（既存ダッシュボードを流用）
- レベル診断（Forms）を改訂レベルに合わせて更新し、**レベル分布の推移**を [ダッシュボード](../dashboard/index.md) で追う（LV2/LV3/エクストラの比率が目標へ寄るか）。
