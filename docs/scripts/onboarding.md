### 研修の全体像（オリエンテーション）＝受講者に共有する前提（スライド「全体像①〜⑤」）
オンボ冒頭で“今回の取組の地図”として5枚を**駆け足**で共有し（**設計書冒頭の正本＝[カリキュラム設計書](../design/curriculum.md)**）、すぐハンズオン（0:18〜）へ入る。話し込みすぎない。

1. **前提：営業プロセスの変革（AI＋へ）** … 依頼ベース→ペーパー往復→**RFPで他社と血みどろ**（後手）から、**調査で課題仮説→議論1〜2回で合意→一気通貫で受注**（先回り）へ。
2. **設計の重心** … 教わる場でなく、**手を動かして毎回“成果物”**を持ち帰る。上達＝毎週の実務反復＋学び合い（わかる→できる→**身につく**）。
3. **レベル定義と卒業条件** … オンボ／LV1／LV2／LV3／エクストラ。各レベルに**卒業条件**（例：LV2＝紙芝居デモを1本、LV3＝案件を1周）。定義の正本は設計書§2。
4. **背骨：営業業務 × AIへの渡し方** … `①調べる→②記録する→③書く→④見せる→⑤回す→⑥作る`。**渡し方の型は1つ＝素材を渡す→指示→AIに作らせる→人が確認・仕上げ**。
5. **3ヶ月スケジュール（途中で変わる可能性あり）** … 混在型（毎回自分で選ぶ／行き来OK）。前半LV1・中盤LV2・後半LV3。**ゴール分布（あくまで想定）＝LV2 1割／LV3 7割／エクストラ 2割**。

> **この回の設計思想（講師向け・最重要）**：知識を教える回ではなく、**全体像（地図）を短く共有 → すぐ手を動かして体感**する回。全体像は駆け足（10分強）で切り上げ、残り時間は受講者がAIを触っている状態を死守。伝えたいメッセージは“言う”より、**やってもらった直後に体験へ紐づけて短く言葉にする**。講師が一方的に話すのは合計15分以内。

### この回で必ず届ける6つのメッセージ
1. AIを使えることはこれからの必須スキル。**学び続け・変わり続ける**行動を取ろう。
2. マインドセットは **「自分の（ルーティンの）仕事をなくす」**。空いた時間を商談など価値ある仕事に回す。
3. ゴールは「わかる」「できる」の先の **「身につく」**（毎週の反復で普段使いに）。
4. これは私が教える場ではない。**一緒に学び・見せ合う**（学び合いが最速）。
5. レベルは段階構成（LV1〜LV3＋エクストラ）。**毎回どのレベルに出るかは自分で選ぶ**（前半LV1→中盤LV2→後半LV3、行き来自由）。
6. **カリキュラムは途中で変わる**（AIの進化が速い＝変化に乗れること自体がスキル）。

### わかる → できる → 身につく の階段（この研修の背骨）
| 段階 | 状態 | どこで効くか |
|---|---|---|
| **わかる** | AIで何ができるか腑に落ちる | オンボーディング前半（全体像） |
| **できる** | 手伝ってもらえば一度できた | オンボーディング後半・L1 |
| **身につく** | 考えなくても普段使いになる | **毎週の反復・宿題（ここが本体）** |
