**ゴール宣言：終わったら、①この取り組み（AI＋への営業変革と3ヶ月の地図）を理解し、②AIで会議録画→議事録を“自力で”整える初回ハンズオンを体験する**

### 研修の全体像（スライド①〜⑤＝受講者に共有する正本）
オンボ冒頭で“今回の取組の地図”として5枚を**駆け足**で共有し（**設計書の正本＝[カリキュラム設計書](../design/curriculum.md)**）、すぐハンズオン（0:16〜）へ入る。話し込みすぎない。

1. **前提：営業プロセスの変革（AI＋へ）** … 依頼ベース→ペーパー往復→**RFPで他社と血みどろ**（後手）から、**調査で課題仮説→議論1〜2回で合意→一気通貫で受注**（先回り）へ。
2. **設計の重心** … 教わる場でなく、**手を動かして毎回“成果物”**を持ち帰る。上達＝毎週の実務反復＋学び合い（わかる→できる→**身につく**）。
3. **レベル定義と卒業条件** … オンボ／LV1／LV2／LV3／エクストラ。各レベルに**卒業条件**（例：LV2＝紙芝居デモを1本、LV3＝案件を1周）。定義の正本は設計書§2。
4. **学びの変化（失敗して修正）** … AI時代は**積極的に失敗しよう／うまくいくまで修正すればいいだけ**。「わかってから動く」より「**やってみる→まちがう→修正する→できる…身につく**」。**わからなくてもいい、まず問いを投げる**。（業務フロー×渡し方の“背骨”はLV1以降＋[設計書§3](../design/curriculum.md)で扱う）
5. **3ヶ月スケジュール（途中で変わる可能性あり）** … 混在型（毎回自分で選ぶ／行き来OK）。前半LV1・中盤LV2・後半LV3。**ゴール分布（あくまで想定）＝LV2 1割／LV3 7割／エクストラ 2割**。

> **設計思想（講師向け）**：全体像①〜⑤は**駆け足で共有 → すぐ手を動かす**。講師が一方的に話すのは合計15分以内、残りは受講者がAIを触っている状態を死守。メッセージは“言う”より、やってもらった直後に体験へ紐づけて短く。

### 事前準備チェック（開始前にTAが確認）
- [ ] 全員 Teams／Copilot にログイン済み
- [ ] 各自、要約に使える**自分のTeams会議録画**を1本用意（例：WeeklyMTG／SIR）。無い人用に**サンプル録画**を配布
- [ ] **Slack**（対話集約）／**Box**（録画・学習記録・BoxNote）／**アンケート(Forms)** の各リンクをスライドから用意
- [ ] TAは開始10分前から待機ルームでログイン未完了者をフォロー

> 時間は目安（60分）。スライド1枚＝1ビートで進める。

### 0:03–0:13　全体像①〜⑤を共有（スライド①〜⑤）
- 🎤（① 自分たちもAI+だよね／前提）「**営業のやり方そのものが変わります。** 従来は依頼ベース→ペーパー往復→**RFPで他社と血みどろ**で後手。AI＋なら**調査で課題仮説→議論1〜2回で合意→一気通貫で受注**。後手から**先回り**へ。」
- 🎤（② 本取り組みの進め方／重心）「この研修は**手を動かして毎回“成果物”を持ち帰る**場。上達は講義だけでなく**毎週の実務利用（反復）＋学び合い**。だから説明は最小限、すぐ触ります。」
- 🎤（③ レベルと卒業条件）「レベルはオンボ→LV1→LV2→LV3→エクストラ。各レベルに**卒業条件**（例：LV2＝デモ1本、**LV3**＝案件1周）。」
- 🎤（④ 学びの変化）「学び方も変わります。**積極的に失敗してOK。うまくいくまで修正すればいいだけ**。『わかってから動く』より『**やってみる→まちがう→修正する**』。**わからなくてもいい、まず問いを投げましょう**。」
- 🎤（⑤ 3ヶ月の流れ）「3ヶ月は**混在型＝毎回自分でレベルを選ぶ**（行き来OK）。前半LV1・中盤LV2・後半LV3。想定分布はLV2 1割／**LV3 7割**／エクストラ 2割。**進め方は途中で変わる可能性あり**——変化に乗るのもスキルです。」

### 0:13–0:16　はじめのお約束（スライド）
- 🎤（マインドセット）「今日の合言葉は **『自分の仕事をなくす』**。ルーティンをAIに渡して、空いた時間で商談を1件増やす、が狙いです。」
- 🎤（安全）「お客様名・個人情報・社外秘は**そのまま入力しない**（伏字か配布データ）。会議は**Teams録画→Copilotで文字起こし**が基本（メモ不要）。」
- 🎤（録画同意・必ず添える）「**お客様との打ち合わせは、録画の前に必ず**『記録のために録画させてください。私たちの記録のためだけに使います』と断って**合意を得てから**にしてください。」

### 0:18–0:25　まずはこちら（録画→要約：Teams操作を画面で真似る）
- 🎤「まず“素材”を用意します。私の画面を真似してください。」（スライドの手順を投影）
  1. Teamsの管理画面を開く
  2. 左ペイン「チャット」から録画された会議を選ぶ（WeeklyMTGでもSIRでもOK）
  3. 録画データをスクロールで探し「要約の表示」をクリック
  4. 新しく開いたウィンドウで「Copilot」をクリック
  5. チャットボットに会議の要約を指示（指示文は自分で考えてみる）
- 🎤「どんな要約が出たか、**ご自身の名前のBoxNote に記録**してください（リンクはスライド／Slack）。」
- 🧑‍🔧TA：✅監視。録画が見つからない人へ即フォロー。
- ⚠つまずき：「録画が無い/見つからない」→ TA「**サンプル録画でOK**。手順の体感が目的です」。

### 0:25–0:35　次にこちら（8項目プロンプトで整形）
- 🎤「次は“指示”。さっきのtranscriptを根拠に、**8項目に整理**させます。以下をCopilotに貼ってください。」
- 💬チャット（コピペ可）：
  ```
  この会議の transcript を根拠に、
  ①会議日付・時間 ②出席者 ③利用された資料 ④議論の論点・議論内容
  ⑤決定事項 ⑥未決事項 ⑦次アクション・担当者候補・期限（テーブル形式）
  ⑧内々の確認・根回し事項
  を整理してください。不明な項目は「要確認」。事実を勝手に作らないこと。
  ```
- 🎤「**中身は“会議そのもの”。AIは形を整えるだけ。** 出たら『もっと短く』『表にして』と重ねてOK。出力は**BoxNoteに記録**。」
- 🧑‍🔧TA：巡回。良い出力・良い指示を2つピック（後の共有用）。わからなければSlackへ。

### 0:35–0:48　ブレイクアウト①：数人で議論（10分議論＋共有）
- 🎤「数人のグループで議論します。①**どういう指示をしたらどんなアウトプットが出たか** ②**どういう指示をすると良さそうか**。10分議論 → その後まとめて共有。」
- 🧑‍🔧TA：各ルーム巡回。詰まりを1分救済。共有役を各グループ1名決めておく。
- 🎤（体現）「正直、私もこの使い方を知ったのは最近です。**見せ合うのが一番速い学び方**。良い指示を全体に広げましょう。」

### 0:48–0:56　まとめ（学びの共有・記録）（スライド）
- 🎤「わかる → できる → 身につく。**今日は“できる”の入口**です。」
