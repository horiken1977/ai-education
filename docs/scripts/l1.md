# L1：初心者セッション台本
### テーマ：「AIに頼んで、仕事の一部を終わらせる」
**今日のゴール宣言：終わったら、Teams会議の“録画”を、整った議事録・上司報告・次アクション一覧に“自力で”整形できる（＝もう会議でメモを取らなくていい）**

> 共通ルール（安全・録画の同意・進め方・記号の意味）は [台本ガイド](index.md) を参照。配布教材は [L1 会議トランスクリプト](../materials/l1-transcript.md)。

> **設計メモ（講師向け）**：L1ではあえて「AIにメール返信を書かせる」等の“生成”タスクを使わない。背景を知らないAIの生成物は修正に時間がかかり、初体験を台無しにするため。
> ここでは **参加者は“素材（会議の中身）”を持ち込み、AIは整形・要約に徹する**＝最も純粋な「人が中身、AIが形」。
> **この会社の前提（最重要）**：受講者には**会議メモを取る習慣がない**。だから「メモを用意して」は高すぎるハードル。**会議はTeamsで録画し、Copilotのトランスクリプト（文字起こし）／会議まとめを“素材”にする**ことを徹底して強調する（メモ取りは不要）。「録画さえすれば3分で議事録・報告・次アクションになる」が今日の感動ポイント。
> ※ハルシネーション対策：トランスクリプトは長く要約時に取り違いが起きうる。重要論点は録画と突き合わせて確認させる／「無いことは“要確認”、事実を作らない」を必ず指示に入れる。
> ※（当面は触れない）本来は重要論点を“その場でタイプ”する方が速く正確だが、できる人が少ないため、まずは録画→トランスクリプトを基本にする。

!!! warning "録画の同意ルール（必ず口頭で伝える）"
    会議は基本 **Teamsで録画** します。ただし **お客様との打ち合わせは、録画の前に必ず断りを入れて合意を得てから** 録画してください。
    言い方の例 →「**記録のために録画させてください。私たちの記録のためだけに使います。**」
    社内会議も録画を基本にしてOK。判断に迷ったら録画前にTA／上長に確認。
