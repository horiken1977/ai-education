# テンプレート集

研修中・研修後にそのまま使えるテンプレートをまとめています。コピーボタンで持ち帰ってください。

## 1. 営業向け プロンプトの基本型（L1で配布）
```
あなたは経験豊富な法人営業です。            ← ①役割
以下の状況です：{背景・顧客情報}            ← ②背景
次をお願いします：{依頼内容}                ← ③依頼
出力形式：{箇条書き／メール文／表 など}      ← ④出力形式
トーン：{丁寧／カジュアル／簡潔}            ← ⑤トーン
```

## 2. 会議トランスクリプト整形（L1の中心プロンプト）
> 会議はメモを取らず、**Teamsで録画 → Copilotで文字起こし（トランスクリプト）／会議まとめ**を作り、それを素材にする。
> お客様との打ち合わせは、録画の前に必ず同意を得る →「記録のために録画させてください。私たちの記録のためだけに使います。」

```
あなたは法人営業の事務サポートです。
次はTeams会議の録画トランスクリプト（文字起こし）です。これを3つの形に整えてください。
① 議事録（日時・出席者・決定事項・論点が分かるように）
② 上司への報告（3〜4行、結論から）
③ 次アクション一覧（やること／担当／期日 の表）
不明な項目は「要確認」と書いてください。事実を勝手に作らないこと。
```

## 3. ディスカッション・ペーパーを作る：伏字プロンプト（L1の中心）
> チャットAIでPPTを作る型＝`調べる→構成→Copilotで伏字プロンプト→Genspark`。**Genspark（社外ツール）には生の社内・顧客情報を渡さない**。社内で使えるCopilotが「一般化した指示書」に変換してから貼る。

**① 構成案をつくる（リサーチ結果を渡して）**
```
上のリサーチをもとに、ディスカッション・ペーパーの構成案（メッセージライン）を作ってください。
スライドごとに「見出し」と「一番言いたいこと（1行）」を並べる。
流れ：表紙 → 現状と課題 → ありたい姿 → 打ち手（当社提案）→ 期待効果 → 次アクション。
各スライドは要点3つ以内。推測・要確認は明記。
```

**② Copilotに Genspark用の伏字プロンプトを作らせる（伏字・一般化）**
```
この構成案から、スライド生成ツール（Genspark）に貼る「PPT生成プロンプト」を作ってください。
・社名/顧客名/個人名/社内数値は【業種】【A社】【約N名】のように伏字・一般化する
・各スライドの見出し・要点(3つ以内)・推奨レイアウト(表紙/箇条書き/図解/数値強調)を指定
・トーンは法人営業の提案向け・簡潔
・そのまま貼れる1ブロックのプロンプトで出力
---
{構成案を貼る}
```

**③ Gensparkで生成→修正**：出てきた伏字プロンプトを貼って生成。「◯枚目を図解に」「効果は数値強調」で修正。
!!! warning "無料枠の前提／伏字チェック"
    Gensparkの無料枠は生成回数・エクスポートに制限あり（**1人1デックが目安**、作り直しは節約し修正で仕上げる）。貼る前に**実社名・顧客名・個人名が残っていないか**を必ず確認する。

## 4. 営業の定常業務 ×テンプレ候補（L2で選ぶ）
| 業務 | テンプレの狙い | 主ツール |
|---|---|---|
| 商談前リサーチ | 企業・業界・キーパーソンの要点整理 | IBM CA |
| 提案書ドラフト | 課題→解決→効果の骨子生成 | IBM CA／Copilot(Word) |
| フォローメール | 会議トランスクリプトから次アクション付き返信 | Copilot(Outlook) |
| 会議議事録 | 文字起こし→要約→ToDo抽出 | Copilot(Teams) |
| 案件管理 | 案件リストの集計・優先度付け | Copilot(Excel) |

### テンプレ雛形（変数を { } で）
```
あなたは{役割}です。
状況：{顧客・案件の背景}
依頼：{してほしいこと}
入力：{貼り付けるデータ}
出力形式：{メール／箇条書き／表}
```

## 4. L3 品質チェックリスト（顧客提出前）
- [ ] 事実・数字の裏取りをしたか（出典確認）
- [ ] 顧客固有情報が正しく反映されているか
- [ ] 社外秘・他社情報の混入がないか
- [ ] 自社のトーン＆ブランドに合っているか
- [ ] 最終判断・責任は人間が持っているか
