# 配布教材：途中の回から参加する人の「開始素材」集（架空）

各回は**1回完結**です。**前回に出ていなくても、その回の“開始素材”さえあれば今日のゴールに到達できます。**
このページは、b/c/d など「前工程の成果物を入口に使う回」のために、**架空企業「みらい製作所」で作った完成済みの中間成果物**をまとめたものです。

- **連続で出ている人**：自分の前回成果物を使ってOK（このページは使わなくてよい）。
- **今日から／久しぶりに来た人**：このページの該当ブロックをコピーして、そのまま今日のワークに入れます。

!!! warning "これは架空データです"
    社名・氏名・数値はすべて架空（みらい製作所＝精密金属部品の中堅メーカー／在庫管理改善の案件）。実在の企業・個人とは関係ありません。
    **社外ツール（Genspark 等）には生の社内/顧客情報を貼らない**。顧客提出物に使うときは事実の裏取り・社外秘の確認を必ず人が行う。

!!! note "a 回（各レベルの入門）の開始素材は別ページ"
    - LV1-a（調べる＋構成）＝ [リサーチ対象サンプル](lv1-research-target.md)
    - LV2-a（Bob入門）＝ [デモ業務テーマ](lv2-demo-theme.md)
    - LV3-a/b（案件の入口）＝ [案件ブリーフ](l3-brief.md)

## LV1-b の開始素材：PPT構成案（＝LV1-a の成果物）

LV1-b（Copilotで伏字発注書 → Genspark でスライド化）の入口。**この構成案を、Copilot に「Genspark用の伏字発注書にして」と渡す**ところから始めます。

```text
[提案PPT 構成案／架空：みらい製作所 在庫管理改善]

タイトル：在庫の“見える化”から始める、みらい製作所の在庫管理改善提案

1. 表紙：在庫の見える化＋発注点アラートで、まず小さく始める
2. 現状の課題：月末棚卸しに 2日×3人／発注が担当者の勘＝属人的／欠品・過剰在庫のリスク
3. あるべき姿：在庫をひと目で把握・発注点を下回ったら自動アラート・基幹連携は段階的に
4. 提案の全体像：Phase1 見える化 → Phase2 発注点アラート → Phase3 基幹連携（可否は要確認）
5. 期待効果：棚卸し工数の削減（月末2日×3人 → 目標◯割減）・欠品リスクの低減（数値は要裏取り）
6. 進め方とスケジュール：小さく試す→現場の声で調整→拡大
7. 体制・次アクション：次回デモで“こんな感じ”を提示（7/17）／品目数・IT予算・意思決定者を確認
```

!!! tip "ファシリ向け"
    LV1-a に出ていた人は、自分が作った構成案を使う。今日から来た人はこのブロックを配布。**伏字（社名・顧客名・生数値）の一般化は、Genspark に貼る前に必ず 1 回挟む**ことを強調。

## LV1-c の開始素材：一気通貫で再利用する「保存済みプロンプト」一式

LV1-c（自案件で一気通貫＋磨き）の入口。本来は LV1-a/b で各自が保存したプロンプトを再利用しますが、**手元に無い人はこの一式をコピー**して、自分の案件（or みらい製作所）で通します。

### ① リサーチ＆構成プロンプト（LV1-a の型）

```text
次の商談前メモをもとに、法人営業の提案準備をしてください。
出力：①業界動向と課題仮説 ②キーパーソン仮説 ③刺さる切り口 ④確認すべき点（裏取りリスト）
そのうえで提案PPTの構成案（見出し＋各ページ要点3つ以内）を作ってください。
条件：不明な点は「要確認」と明記し、事実を作らない。数字・固有名詞は仮説として扱う。
---
{商談前メモを貼る（自案件 or 上のリサーチ対象サンプル）}
```

### ② 伏字発注書プロンプト（LV1-b の型）

```text
次の提案の構成案から、スライド生成ツール（Genspark）に貼る「PPT生成プロンプト」を作ってください。
条件：
・社名・顧客名・個人名・具体的な社内数値は【業種】【A社】【約N名】のように伏字／一般化する
・各スライドの見出しと要点（3つ以内）、推奨レイアウト（表紙/箇条書き/図解/数値強調）を指定
・トーンは法人営業の提案向け・簡潔
出力はそのまま貼れる1ブロックのプロンプトにしてください。
---
{構成案を貼る（自案件 or 上のPPT構成案サンプル）}
```

## LV2-b の開始素材：要件3点シート（＋デモ業務テーマ）

LV2-b（自分の案件でデモ制作）の入口。LV2-a のデモが手元に無くても、**要件を「誰が・何の画面で・何ができる」の3点で言葉にすれば、Bob にその場で発注できます。** 題材が思いつかない人は [デモ業務テーマ](lv2-demo-theme.md) を流用。

```text
[デモ要件シート／架空：みらい製作所 在庫管理デモ]

・誰が：倉庫の在庫担当者
・何の画面で：在庫一覧／入荷の登録フォーム／発注点アラート
・何ができる：在庫をひと目で見る・入荷を登録する・少なくなった品目を赤く表示する

Bobへの一言：
在庫管理業務の、担当者が使うデモ画面を作って。一覧と「登録」ボタン、在庫が少ない品目を赤く表示する感じで。まずはざっくりでOK。
```

## LV2-c の開始素材：磨く対象のデモ（再現手順＋各画面の一言）

LV2-c（磨き・見せ方）の入口。磨く対象のデモが手元に無い人は、**この再現手順で Bob に一度作り直してから**、見せ方の練習に入ります。

```text
[磨き対象デモ／架空：みらい製作所 在庫管理デモ]

再現の一言（Bobに貼る）：
在庫管理のデモ画面。在庫一覧（品目・数量・発注点）、入荷登録フォーム、発注点を下回った品目を赤く表示。まずは紙芝居レベルでOK。

各画面が解く課題（トークの下敷き＝1文キャプション）：
・在庫一覧 … 「今、何がどれだけあるか」を探さずに一目で分かる
・入荷登録 … 入荷のたびに手作業のエクセル更新をなくす
・アラート … 発注忘れ／欠品を、勘に頼らず気づける
```

## LV3-c の開始素材：提案骨子（＝LV3-b の成果物）

LV3-c（デモ→提案資料）の入口。LV3-b の骨子が手元に無い人はこれを使います。案件の詳細は [案件ブリーフ](l3-brief.md) 参照。

```text
[提案骨子／架空：みらい製作所 在庫管理改善]

・一言サマリー（エレベーターピッチ）：
  「月末2日×3人の棚卸しと、勘に頼る発注を、“見える化＋発注点アラート”で小さく始めて減らします。」

・課題：在庫がエクセル手作業／棚卸し 月末2日×3人／発注が属人的で欠品・過剰在庫リスク
・解決：在庫の見える化 → 発注点アラート → （段階的に）基幹連携
・効果：棚卸し工数の削減・欠品リスク低減（数値は要裏取り）・発注の標準化
・次アクション：次回（7/17）にデモ＋1枚サマリーを提示／品目数・IT予算・意思決定者を確認
```

## LV3-d の開始素材：提案資料一式（＝LV3-b/c の成果物）

LV3-d（フォロー→クロージング＋品質）の入口。一周の後半を繋ぐ材料。**b/c に出ていなくても、この一式を“前半の成果”として今日の後半に接続**できます。

```text
[提案資料一式／架空：みらい製作所 在庫管理改善]

■ 提案骨子：上の LV3-c 開始素材のとおり（見える化→発注点アラート→段階的に基幹連携）
■ デモ：在庫一覧／入荷登録／発注点アラートの紙芝居（LV3-c で作成）
■ 提案書ドラフトの要点：現状課題 → 解決の全体像 → Phase設計 → 効果（要裏取り）→ 進め方 → 次アクション
■ 想定Q&A：
  ・Q「基幹システムと連携できるの？」→ A「まず見える化から。連携は可否を確認して段階的に」
  ・Q「現場の負担が増えない？」→ A「入力は最小限、まず一覧を見るだけでも効果」
■ 商談ステータス：初回ヒアリング済み → 7/17 デモ提示予定。予算感・意思決定者は未確認。
```

!!! danger "顧客に出す前の品質チェック（人が最終判断）"
    事実・数字が裏取り済みか／固有名詞・社外秘が混ざっていないか／AIが作った“それっぽい数値”を鵜呑みにしていないか。**最終責任は人間**。
