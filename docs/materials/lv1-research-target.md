# 配布教材：リサーチ対象サンプル（架空企業）

LV1-a（中身をつくる＝調べる＋構成）の写経ワークで使う、**架空企業の概要メモ**です。
参加者はこれを ICA / Copilot に渡して、業界動向・課題仮説・キーパーソン仮説・刺さる切り口・**確認すべき点**を出させ、続けて**ディスカッション・ペーパーの構成案**まで作ります（次回 LV1-b でGensparkでスライド化する素材になる）。

!!! warning "これは架空データです"
    社名・数値はすべて架空。実在の企業とは関係ありません。リサーチは**公開情報のみ**を扱い、社外秘・未公開情報は入力しない。AIの出力は“仮説の下書き”として、数字・固有名詞・最新動向は必ず人が裏取りする。

!!! tip "わざと情報を少なめにしています"
    全部は分からない前提。AIに「不明な点は要確認、推測は推測と明記」と指示し、**確認すべき点**を出させる練習に使ってください。

## サンプル（このブロックをコピーして使う）

```text
[商談前リサーチ対象メモ／架空]

会社名：みらい製作所（架空）
業種：精密金属部品の製造（中堅メーカー）
事業内容：自動車・産業機械向けの金属加工部品。設計から量産まで。
規模感：従業員 約300名 ／ 拠点 本社工場＋地方に第2工場（新設の話あり・未確認）
最近の動き（公開情報想定）：受注は回復傾向との業界紙コメント。人手不足・原材料高が課題と発言。
商談の接点：購買部（コスト・在庫）、情報システム部（基幹システム）
これまでに分かっていること：在庫管理が手作業（エクセル）で、棚卸しに月末2日×3人。発注が担当者の勘に依存。
当社が提供できるもの：在庫の見える化／発注点アラート／基幹システム連携（連携可否は要確認）

不明・未確認：年商、品目数、IT予算、意思決定者、競合の有無
```

!!! note "このサンプルから作れるもの（ファシリ向け）"
    AIに次の4点を出させる：
    
    1. **業界動向と課題仮説**：人手不足・原材料高の中で、在庫の手作業と属人的発注はコスト・欠品リスク。
    2. **キーパーソン仮説**：購買部＝コスト/在庫の痛み、情シス＝連携と運用、意思決定は上長（前回「田村」氏の名前あり＝要確認）。
    3. **刺さる切り口**：棚卸し工数（月末2日×3人）の削減と、発注の標準化。まず見える化から小さく。
    4. **確認すべき点**：基幹システムの種類・連携可否／品目数／IT予算／意思決定プロセス／競合提案の有無。
