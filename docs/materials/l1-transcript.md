# 配布教材：Teams会議トランスクリプト サンプル（架空）

オンボーディング／LV1 の写経ワークで使う、**架空のTeams会議録画トランスクリプト（文字起こし）**です。
参加者はこれを Copilot に貼り付けて、議事録・上司報告・次アクション一覧に整形します。

!!! tip "本番の進め方（受講者へ）"
    会議でメモは取らない。**会議をTeamsで録画 → Copilotでトランスクリプト／会議まとめを生成** → それをAIの素材にする。
    お客様との打ち合わせは、録画の前に必ず同意を得る →「**記録のために録画させてください。私たちの記録のためだけに使います。**」合意を得てから録画。

!!! warning "これは架空データです"
    社名・氏名・数値はすべて架空。実在の企業・個人とは関係ありません。研修以外で実データを入れる場合は社外秘・個人情報に注意。

## サンプル（このブロックをコピーして使う）

```text
[Teams会議 文字起こし]  2026-07-03（金）14:00-14:35  オンライン

山田（弊社・営業）: お時間ありがとうございます。弊社の山田です。今日はよろしくお願いします。録画はさせていただいてますが、私たちの記録用だけに使いますので。
鈴木（みらい製作所・購買部）: はい、大丈夫です。購買の鈴木です。
高橋（みらい製作所・情報システム）: 情シスの高橋です。よろしくお願いします。
山田: 前回いただいた、在庫管理がかなり手作業で大変、というお話をもう少し伺えればと。
鈴木: そうなんですよ。今エクセルで各倉庫の在庫を管理してて、月末の棚卸しのたびに、えーと、3人がかりで丸2日くらいかかってます。
高橋: 入力ミスも多くて。発注のタイミングも担当者の勘に頼ってる感じですね。
山田: なるほど。月末2日、3人ぶん。発注も属人的、と。…ちなみに対象の品目数ってどのくらいですか。
鈴木: 正確には今すぐ出ないんですが、たぶん2,000点くらい…ちょっと確認します。
山田: 助かります。であれば、まず在庫の見える化と、発注点のアラートあたりから入るのが効きそうですね。
高橋: それ欲しいです。あと既存の基幹システムと連携できるかが気になります。
山田: そこは持ち帰って確認させてください。次回、御社の業務に寄せた簡単なデモ画面をお見せできればと思います。
鈴木: いいですね。上長の田村にも見せたいので、できれば資料も一枚あると。
山田: 承知しました。デモと、1枚サマリーを用意します。日程は来週後半でいかがでしょう。
鈴木: 17日の金曜午後なら空いてます。
山田: では7/17（金）15時で仮で押さえます。品目数だけ事前に共有いただけると、デモがより具体的になります。
鈴木: 了解です。確認して水曜までに送ります。
山田: ありがとうございます。本日はここまでで。失礼します。
```

!!! note "このサンプルから作れるもの（ファシリ向け）"
    - **議事録**：日時・出席者・現状課題（在庫管理が手作業／月末2日×3人／発注が属人的）・決定事項（次回デモ＋1枚資料）・次回 7/17(金)15:00
    - **上司報告（3行）**：結論＝次回デモで前進。課題は在庫の手作業と属人的発注。次アクション＝デモ準備・基幹連携の確認。
    - **次アクション一覧**：①基幹システム連携の可否を確認（山田）②デモ＋1枚資料を作成（山田）③品目数を共有（鈴木・水曜まで）。「品目数=約2,000（要確認）」のように**未確定は要確認**で残せる。
