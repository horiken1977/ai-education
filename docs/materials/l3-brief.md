# 配布教材：案件ブリーフ サンプル（架空）

LV3（使い分け）の案件演習で使う、**架空の案件ブリーフ一式**です。
参加者はこれを使い、ICA/Copilot/Bob を使い分けて リサーチ→提案→デモ→フォロー を一周します。

!!! warning "これは架空データです"
    社名・氏名・数値はすべて架空。実在の企業・個人とは関係ありません。顧客提出物に使う場合は事実の裏取り・社外秘の確認を必ず行う。

## 案件ブリーフ（このブロックをコピーして使う）

```text
[案件ブリーフ／架空：みらい製作所 在庫管理改善]

■ 顧客概要
・会社名：みらい製作所（架空）／精密金属部品の中堅メーカー
・規模：従業員 約300名、本社工場＋第2工場（新設の話あり・未確認）
・接点：購買部 鈴木氏、情報システム部 高橋氏、上長 田村氏（決裁者と推定・要確認）

■ 業界・背景
・人手不足と原材料高でコスト圧力。製造現場のDXは関心高いが、現場負担を嫌う傾向。

■ 課題（ヒアリング済み）
・在庫管理がエクセルの手作業。月末棚卸しに 2日×3人。
・発注タイミングが担当者の勘＝属人的。欠品/過剰在庫のリスク。
・基幹システムとの連携可否が不明（情シスの関心事）。

■ 今回の提案テーマ
・在庫の見える化＋発注点アラートから小さく始める。基幹連携は段階的に。

■ 予算・時期
・予算感は未確認。次年度予算の検討時期に入る前に方向性を固めたい意向（推定）。

■ 競合・その他
・他社提案の有無は不明（要確認）。現場の使いやすさを最重視。

■ これまでの経緯／次のマイルストン
・初回ヒアリング済み → 次回 7/17(金)15:00 にデモ＋1枚サマリーを提示予定。
・事前に品目数（約2,000・要確認）を共有してもらう約束。
```

!!! note "この案件で一周する流れ（ファシリ向け）"
    どの業務に・どのAIかを意識して使い分ける。

    - **① リサーチ（ICA/Copilot）**：業界・課題・キーパーソンの整理と、確認すべき点の洗い出し。
    - **② 提案骨子（Copilot）**：課題→解決→効果→次アクション。「見える化から小さく」のストーリー。
    - **③ デモ（Bob）**：在庫一覧＋発注点アラートの紙芝居デモ（[LV2 デモ業務テーマ](lv2-demo-theme.md) テーマ①と連動）。
    - **④ フォロー（Copilot）**：デモ後のお礼＋次アクション、田村氏向け1枚サマリー、想定Q&A（基幹連携・価格）。
    - **品質チェック**：品目数・予算・決裁者など**未確認は「要確認」**のまま顧客提出前にレビュー。
