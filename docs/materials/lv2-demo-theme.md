# 配布教材：デモ業務テーマ（架空）

LV2-a（Bob入門＋初めての1本）の写経ワークで使う、**架空の業務シナリオ**です。
参加者はこれを Bob に渡して、「◯◯業務のデモを作って」と話しかけ、紙芝居レベルの画面プロトを作ります。

!!! warning "これは架空データです"
    お客様の**実データ・社外秘は入れない**。架空データ・架空業務で作る。
    完璧なアプリは不要。**紙芝居レベル**で「こんな感じ」を体感させるのが目的。見せ方は画面提示でOK。

## サンプル業務テーマ（どれか1つ選んでBobに渡す）

### テーマ① 在庫管理（架空・みらい製作所）
- 使う人：倉庫の在庫担当者
- 画面：在庫一覧／入荷の登録フォーム／発注点を下回ったらアラート
- やりたいこと：在庫をひと目で見る・入荷を登録する・少なくなった品目に気づく
- **Bobへの一言**：`在庫管理業務の、担当者が使うデモ画面を作って。一覧と「登録」ボタン、在庫が少ない品目を赤く表示する感じで。まずはざっくりでOK。`

### テーマ② 問い合わせ受付（架空）
- 使う人：カスタマーサポート担当
- 画面：問い合わせ一覧／新規受付フォーム／ステータス（未対応・対応中・完了）
- やりたいこと：問い合わせを記録し、対応状況を管理する
- **Bobへの一言**：`問い合わせ受付のデモ画面を作って。一覧と新規受付フォーム、ステータスを切り替えられる感じで。`

### テーマ③ 案件進捗ボード（架空）
- 使う人：営業チーム
- 画面：案件カードを「リード・商談・提案・受注」の列で並べる
- やりたいこと：案件の進み具合をボードで見える化する
- **Bobへの一言**：`営業の案件進捗ボードのデモを作って。リード・商談・提案・受注の列に案件カードを並べる感じで。`

!!! note "ファシリ向けメモ"
    最初は **テーマ①** を全員共通でやると進めやすい。慣れたら自分の業務に置き換える（BO①）。
    「凝りすぎ」を止めるのがコツ＝**完璧より“見せられる”**。動かない所は本番ではこうなりますと口頭で補う。
