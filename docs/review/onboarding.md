# オンボーディングの学び

初回オンボーディング（全員必須）の振り返りです。**会議の録画から Copilot で議事録・要約を作る**ハンズオンを通じて、AIへの指示の出し方とマインドセットを体験しました。以下は、その場で**議論になったこと**と**学びになったこと**をまとめたものです（発言者は記載しません）。

!!! note "この回のねらい"
    目的は「議事録を作れるようになること」ではありません。**AIをどう使い、どう指示を出すか**を、自分の手を動かして体験するのがねらいです。「わかる → できる → 身につく」の“できるの入り口”。

## この回のテーマ
- なぜ営業自身が「AIプラス」になるのか（営業プロセスの変革の前提）。
- 会議録画 → Copilot で議事録・要約：AIへの指示の基本を体験する。
- 3ヶ月の進め方（LV1 → LV2・LV3 → エクストラ）とお約束。

## 学び①：営業プロセスのどこにAIが効くか
- 営業プロセスをざっくり **①入口・課題発見 → ②アプローチ → ③マテリアル準備・ディスカッション → ④合意形成 → ⑤受注** と捉えると、AIが特に効くのは **①入口** と **③マテリアル**。
- **①入口**：AIでリサーチして顧客課題の仮説を立て、自分から立案する。仮説の精度が上がるほど刺さる初回ディスカッションになり、**「RFPを出してもらう」から「RFPを回避する」動き**へ変えられる。
- **③マテリアル**：これからはパワポを作り込むより、**デモや画面（HTML/Web）で実イメージを見せて合意する**方が速い場面が増える。
- 自分たち自身がAIプラスになる＝**「これは付加価値を生んでいるか？」と疑問に感じる作業は積極的にAIへ渡してなくす**、というマインドで臨む。

## 学び②：会議録画から議事録・要約を作る（ハンズオンの要点）
- Teams の録画から Copilot の「要約を表示」やトランスクリプトを起点に、**自分の観点で指示を出して**要約する。
- **Copilotが自動生成する要約と、自分が指示して出す要約は別物**。「どんな観点でまとめてほしいか」を自分から指示することが肝心。
- **先頭に「この会議のトランスクリプトを根拠に」と一文入れる**と、前のやり取りを引きずらない（文脈の混線を防ぐ）。この一文が無いと前の指示を引きずることがある＝地味だが重要。
- トランスクリプトをそのまま貼るのではなく、**ひと手間（整形の指示）を加えるだけ**で、顧客コールメモや議事録としての活用度が大きく上がる。

## 学び③：プロンプトの「型」は育てて貯める
- 議事録の**型・粒度は人や用途で違う**。報告相手・目的によって分量や書き方を変える必要があり、**何度もやり直して自分バージョンに育てる**のが前提。
- **プロンプトそのものをAIに作らせる**のが効く：「この議事録を今後も同じ品質で出すには、どう指示すればいい？」と聞くと、AIが**おすすめの指示文を提案**してくれる。さらに「役員向けの短縮版が必要なら再整理しますが？」のように**AIが逆提案**してくることもある。
- 良かった指示文は**自分のノートにストックし、次回は最初から入れる**。やり取りの回数が減り、品質が安定する。
- これは後の Bob 活用にもつながる「**コンテキスト（指示や定型作業の塊）を貯める**」という発想。
- 長い会議（1時間規模）は**テキスト量が多すぎて一度に処理しきれない**ことがある。分割し「1回目はまだ続きがあるので暫定整理はしないで」等と指示すると扱いやすい。

## 学び④：AIとの向き合い方（マインドセット）
- **わかってから動く、ではなく、わからなくてもまず投げてみる**。ピンとこなければ指示を修正する／やり方を変える。**失敗を歓迎**する。
- AIは「**とても優秀だが、自社のお作法・顧客・背景を知らない新人**」だと捉える。思考力は高く24時間動くので、**背景と指示をきちんと与える**ほど力を発揮する。
- 想定外の答えが返ったら、「自分はこう考えている」「抽象度を上げて／具体性を上げて／この観点で」と**対話を重ねて寄せていく**。

## つまずき・よくある論点
- 「AIに要約させると**自分の思考力が落ちないか**」という懸念が出た。→ 議事録は“聞いたことを反芻しながらまとめる”ことに意味がある面もある。**議事録づくりはあくまでAIの使い方を学ぶ“例”**であり、論点を自分で整理してメモした方が速く頭に残る場合もある＝**ケースバイケース**。試行錯誤も大切だが、時間がもったいない時はAIに聞くのも正解、というバランス感覚を持つ。
- 「**思ったイメージをどう指示に表現するか**が難しい」という声。→ うまくいった型を共有し合うこと、そして前述の「指示文自体をAIに作らせる」ことが助けになる。
- Copilotの管理画面や画面共有まわりの操作でつまずく人も。→ 恥ずかしがらず声を上げ、できる人が画面共有して助け合うのが早い。

## 学び⑤：発展的な使い方（共有された工夫）
- **提案資料づくり**：万能な“決定版”ツールは無い。数枚のピッチ／ライトニングトークなら画像生成込みの生成AIで見栄えのよい資料が作れる。社内標準の作り込みはチャットAI単体だと難易度が高い。
- **外部の資料生成AIも、機密をマスキングすれば使える**。有効だった二段構え：**①Copilotに「社外秘を抜いた差し支えないプロンプト」を作らせる → ②それを外部の資料生成AIに渡して骨子を作る → ③社外秘情報は後からCopilotで入れる**。これで提案骨子が短時間で用意できたという共有があった。ここでも肝は「**資料を作る指示文そのものをAIに作らせる**」こと。
- 生成された資料は**編集可能な形式で出力**され、社内テンプレートの“ガワ”だけ渡すと社内風の体裁にしてくれたり、しゃべり原稿（5分／3分／1分版）まで作れたりする。ただし**トークン上限・課金には注意**。
- ブラウザ操作やデータ収集まで自動化する強力な使い方もあるが、**AIを野放しにするとガバナンスが効かなくなる**。ローカル完結なら影響は小さい。ここまで体験すると、**AIガバナンスの重要性を“自分ごと”として実感**できる。

## 復習ポイント（次に活かす）
- まずは**業務で1回使う**。説明を待たず、雑でいいので投げてみる。
- 要約・整形の指示は**「トランスクリプトを根拠に」＋観点指定**で出す。
- **うまくいった指示文はノートに貯める**。困ったら「どう指示すればいい？」とAI自身に聞く。
- 外部ツールを使うときは**機密のマスキング**を徹底し、社外秘は社内ツール側で扱う。
- 学び・プロンプトは各自の記録先に残し、気づきは共有の場に投げて**学び合う**。
